# Launchia (ランシア) プロジェクト 発展的拡張案

## 1. 概要
本ドキュメントは、既存の「WebForm（Webパーツ）」の技術基盤と、そこから派生した「Launchia（ウェイトリスト・マーケットプレイス）」のコンセプトを融合・発展させるための具体的な拡張アイデアを提案するものです。

Launchiaを単なる「順番待ちリスト」から、**「ローンチ前のコミュニティ形成と熱量醸成のプラットフォーム」**へと進化させることを目的とします。

---

## 2. 拡張アイデア：プロダクト機能面

### 2.1 バイラル・エンジンの高度化 (Advanced Viral Engine)
現在の「友人を招待すると順番が繰り上がる」仕組みをさらに強化します。
- **マイルストーン報酬:** 5人招待で「クローズドベータ参加権」、10人招待で「生涯無料プラン」など、段階的なインセンティブ設定機能。
- **ソーシャル・プルーフのリアルタイム表示:** 埋め込みパーツ内に「現在 1,234 人が待機中」「3分前に 〇〇 さんが登録しました」といった動的な通知を表示し、FOMO（取り残される恐怖）を刺激。

### 2.2 開発者向けインサイトの強化 (Developer Analytics)
単なる登録者リストではなく、ユーザーの「熱量」を可視化します。
- **熱量スコア:** ユーザーが自分の順位を確認した回数や、招待リンクのクリック率などを元に、アクティブなユーザーを抽出。
- **属性分析:** どのLP、どのキャンペーン（solution-id）から流入したユーザーが最も紹介貢献度が高いかを分析可能に。

### 2.3 ユーザー体験のゲーミフィケーション (Gamification)
ユーザーが「待つこと自体」を楽しめる仕組みを構築します。
- **バッジ・称号システム:** 「Early Adopter」「Top 1% Referrer」などのバッジをマイページで付与。
- **シークレット・プレビュー:** 順位が高いユーザーのみがアクセスできる、開発中の画面や限定動画の公開機能。

---

## 3. 拡張アイデア：技術・アーキテクチャ面

### 3.1 埋め込みパーツのカスタマイズ性向上
開発者が自社LPのデザインに合わせ、ノーコードでパーツの外見を調整できる仕組み。
- **テーマ注入機能:** CSS Variablesを使用した、`<launchia-widget primary-color="#ff0000" border-radius="8px">` のような動的なスタイリング。
- **ヘッドレス・モード:** UIを提供せず、ロジック（登録・順位取得）のみをAPI経由で提供し、開発者が完全に独自のUIを構築できるSDK。

### 3.2 外部プラットフォーム連携 (Integrations)
コミュニティ形成を加速させるための外部連携。
- **Discord/Slack 連携:** ウェイトリストへの登録やランクアップを、開発者のコミュニティチャンネルに自動通知。
- **X (Twitter) 連携:** 登録完了画面から、現在の順位をワンクリックでシェアできる「シェア用画像（OGP）」の自動生成。

### 3.3 Web3/Wallet 接続対応 (dApp 対応)
次世代アプリの需要に応える機能。
- **Wallet Auth:** メールアドレスの代わりに、Wallet（Metamask等）を接続してウェイトリストに参加。
- **NFT パス:** 特定のNFTを所有しているユーザーに、自動的に上位の順位や「VIPステータス」を付与するゲート機能。

---

## 4. 成長戦略 (Go-to-Market Strategy)

### 4.1 「Dogfooding」の実施
Launchia 自体のウェイトリストを、Launchia の Web パーツを使って構築します。これにより、最初のユーザーに対して「Launchia を使うとどのような体験が得られるか」を直接提示します。

### 4.2 ディレクトリ・ポータルとしての強化
単なるツール（SaaS）に留まらず、Product Hunt のように「次に流行るアプリを探しに来る場所」としてのブランドを確立します。
- **「今週の注目プロジェクト」:** 登録者数の増加率が高いプロジェクトをピックアップしてニュースレターで配信。
