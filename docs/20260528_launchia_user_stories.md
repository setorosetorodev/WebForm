# Launchia の UX — 「気づきからハッピーまで」3 人のストーリー

Phase 1 MVP の機能は、実際に使う人にとってどんな体験になるのか。
要件定義書だけだと「機能の羅列」になってしまうので、
3 人の登場人物の物語で書いてみました。

---

## 1. 一般ユーザー：あゆみさん（28歳・デザイナー）の場合

### 気づき
ある日、X のタイムラインで友達がポストしていた

> 「だすね」が楽しみすぎる…

というツイートが目に入る。「だすね？ なにそれ気になる」と、リンクをタップする。

### 登録
dasune.net を開くと、おしゃれな LP。ヒーローのすぐ下に
こんなフォームが現れる：

```
   リリースされたら一番に使えます。

   [ メアド入力 ___________________ ]   [ ウェイトリストに登録 ]
```

メアドを入れて、ぽちっ。

### 確認待ち
画面が切り替わる：

```
   📧 確認メールを送信しました
   メール内のリンクをクリックして、登録を完了してください。
   クリック後、あなたの順位を確認できます。
```

「あー、確認メールね、よくあるやつ」とメールを開く。

### 達成感（クリックで完了）
メールが届いている。

> 【だすね】ウェイトリスト登録の確認をお願いします
> 登録を完了するには、下記のボタンをクリックしてください。
> [ 登録を完了する ]

ボタンをタップ → ブラウザが開いて：

```
   🎉 登録が完了しました！
   ウェイトリストへの参加が確定しました。

   現在の順位
   42 番目
   （総登録者 1,200 人）
```

「お、私けっこう早いほうじゃん」と、ちょっと嬉しくなる。
**「メアド入れた瞬間ではなく、自分で『完了する』ボタンを押した瞬間に確定した」という小さな儀式感** が、達成感を強めている。

### 思い出させてくれる仕掛け
URL は同じものでブックマークしておく。3 週間後、ふと「あのアプリどうなったかな」と思い出してアクセス：

```
   あなたは 42 番目 です。
   （総登録者 1,200 人）
```

「私、まだまだ古参組じゃん」とニヤリ。
「もしかして 1000 人のうち上位 5% かも」と、特別感が積み上がっていく。

### リリース当日（※ これは Phase 2 の話）
ある朝メールが届く。

> だすねがリリースされました。
> 先行登録者の皆様、お待たせしました。

→ あゆみさんがハッピーになったポイント：
**「自分は他の人より早く気づいてた」という承認感、特別感、希少性**

## 2. 開発者：健太さん（32歳・副業エンジニア）の場合

健太さんは個人開発で「みえまる」というプロダクトを作っている。
プログラミングは得意だが、マーケティングは正直よくわからない。

### 悩み
「リリースしてからユーザー集めても遅い気がする。
でも、メアド収集の仕組み自作するのも面倒だし、
Mailchimp は機能多すぎる…」

### 気づき
別の個人開発者と話していたとき、

> Launchia 使ったらいいよ、無料で簡単だし

と教えられる。`launchia.net` を訪問する。

### サインアップ（招待制）
招待コード `LCHA-XXXX-XXXX` をもらってメアドを入力。
すぐに Magic Link が届く → クリックで瞬時にログイン完了。

「お、パスワード作らなくていいんだ。ラク」

### プロジェクト作成
「新規プロジェクト」をクリック：

- 名前：「みえまる」
- slug：「mienomaru」
- 説明：200 文字くらい書く
- カバー画像：適当な絵をアップ

ここで画面に 2 つの選択肢が出てくる：

```
   [ ✓ ] LP 埋め込みモード（自社サイトに埋め込む）
   [ ✓ ] アイデア公開モード（launchia.net/p/mienomaru で公開）
```

健太さんはまだ LP がないから、**アイデア公開モード**だけ ON にする。
「お、LP 作る前から始められるじゃん」

### 最初の登録者
公開して 1 時間後、Twitter で「こんなの考えてます」とリンクをシェア。
管理画面に戻ると、登録者数が「3」になっている。
「**やった、初めての登録者**だ」

### 進捗の楽しみ
1 週間で登録者 50 人。「うわー、需要あるかも」とテンションが上がる。

LP を作り終わった頃、管理画面で「LP 埋め込みも ON」にスイッチ。
ワンクリックで埋め込みコードをコピー → 自社サイトにペースト。
LP からの登録も、アイデアページからの登録も、同じウェイトリストに溜まる。

「LP 完成しても、アイデアモードで集めた人を引き継げる」

→ 健太さんが得たもの：
**「LP なくても始められる」「アイデアフェーズから検証できる」という安心感**

## 3. 運営者：瀬戸口さん（あなた）の場合

### ステップ 1：ドッグフーディングで実証する
「だすね」LP を作る前に、まず `launchia.net/p/dasune` を **アイデアページ** として公開。
コンセプトを語りながら、登録者を集めていく。

1 ヶ月で 200 人登録 →
**「Launchia は機能するぞ」** という実証データを手に入れる。

### ステップ 2：他の開発者に拡げる
限定 5 社モニターを募集して、個別カスタマイズ支援を伴走。
（短期収益：15〜30 万円/月）

成功事例「**だすね**」を公開ケーススタディに。
Launchia のトップページで「Powered by Launchia: dasune.net」を控えめに紹介。

### ステップ 3：本格展開（Phase 2 以降）
- 招待リンク機能 → バイラル拡散
- Founder's Edition プレセール → 開発資金プレ回収
- マーケットプレイス公開 → 「次に流行る」を発見する場所に

→ 運営者が得たもの：
**「Launchia で実際にプロジェクトが伸びた」事実 = 営業の最強カード**

## このストーリーから読み取れる、Phase 1 MVP の「外せないツボ」

| ツボ | なぜ大事 | 該当機能 |
| :--- | :--- | :--- |
| 「登録を完了する」クリックで初めて順位が出る | ゴミアドレスを弾く + 「自分で確定した」儀式感 | ダブルオプトイン (`confirmed_at`) |
| 確定後に「🎉 登録が完了しました！」+「あなたは N 番目」 | あゆみさんの達成感の核 | `/r/<token>` 初回時の歓迎メッセージ |
| メールに順位確認 URL を残す | 「思い出してくれる」きっかけ | rank_token マジックリンク |
| LP がなくても始められる | 健太さんの「ハードル」を消す | アイデアモード `/p/<slug>` |
| 同じプロジェクトで両モード持てる | LP ができても引っ越し不要 | embed + idea の 2 フラグ |
| マルチテナント設計 | 運営者の事業拡大の素地 | `project_id` 分離 |
| Magic Link 認証 | サインアップの摩擦ゼロ | パスワード機能なし |

## Phase 1 で「あえて作らないもの」と、その代わり何が起きるか

| あえて作らない | 代わりに起きること |
| :--- | :--- |
| 招待リンクによる順位繰り上げ | あゆみさんは「自分の順位が固定」だが、まだ「自慢する文化」は生まれない（Phase 1.5 で開放） |
| ドメイン MX チェック / SMTP 存在確認 | 健太さんのプロジェクトに、メアドの「形式は合ってるけど実在しないドメイン」が一部入る可能性。ダブルオプトインで「届いたメアド」かは保証される（Phase 2 で MX チェック検討） |
| FOMO の「現在 1,234 人登録中」表示 | ページがシンプルすぎて寂しい瞬間がある（Phase 2 で追加） |
| 開発者へのリアルタイム通知 | 健太さんは管理画面を自分で見にくる必要がある（毎日 OK） |
| カスタムドメインからの送信 | メールは全部 `noreply@launchia.net` から（信頼性は SPF/DKIM で担保） |
| マーケットプレイス一覧 | Launchia 内部で他プロジェクトを「探す」機能はない（Phase 3） |

→ あえて作らないことで、Phase 1 のコア体験 ―
**「メアド入れたら順位が出る、後で見られる」** ―
を、最短で確実に届けることに集中する。
