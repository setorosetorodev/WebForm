# Launchia トップ LP — コンテンツ仕様（クロ担当）

トップ `/`（launchia.net）の LP に載せる**中身**を定義する。デザイン/トンマナは Stitch 側で詰め、
このコピーと構成を Stitch に渡して再生成 → 実装する想定。**実在しない機能は書かない**のが大原則。

- 対象: **開発者メイン**（プロジェクト運営者）。エンドユーザー向けは“従”で一言だけ。
- トーン: 元気・ワクワク・前向き、開発者に親しみ。誇張・架空機能なし。直訳調を避けた自然な日本語。
- 制約: **サインアップは招待制**。主 CTA は「無料登録」ではなく **ログイン（招待コードをお持ちの方）**。

## 実機能の線引き（Stitch が勝手に足さないように）

### ✅ 今うたってよい（実在する）
- ウェイトリスト（プロジェクト）の作成・管理（管理画面）
- 埋め込みウィジェット（Web Component を数行）／ホスト型アイデアページ `launchia.net/p/<slug>`
- メール登録＋**ダブルオプトイン**（確認メール）／到達性（SPF・DKIM・DMARC 対応）
- 登録者が**自分の順位**を確認できる（`/r/<token>`）。順位は確認済みエントリの動的ランキング
- 登録者・順位・件数のダッシュボード管理
- 招待制サインアップ／登録解除

- GitHub 等のソーシャル認証（今はメール Magic Link）
- 限定バッジ／先行アクセス権の段階ロールアウト
- Webhook／Slack・Discord 連携
- 「無料で登録」表現、実数の参加者数（「1,234人」等）、「5分で完了」等の断定

### 🗺️ 将来検討（LP には今出さない／メモ）

## セクション構成 & コピー（実装の正）

### 0. ヘッダー
- ロゴ: `Launchia`
- 右: **ログイン**（→ `/login`）

### 1. ヒーロー
- 見出し（第1候補）: **公開前から、「欲しい人」を集めておく。**
  - 代替案: 「リリース前から、"待ち遠しい"をつくる。」／「作る前に、欲しい人と出会う。」
- サブ: Launchia は、リリース前のプロダクトのためのウェイトリスト。自社サイトに数行貼るだけで、メール登録と順位表示が動き出します。
- 主 CTA: **ログインして始める →**（→ `/login`）
- CTA 補足: 招待コードをお持ちの方（現在は招待制）

### 2. 使い方は 3 ステップ
1. **プロジェクトを作成** — アプリ名と説明を登録するだけ。すぐにウェイトリストが開きます。
2. **サイトに設置 / 公開** — ウィジェットを数行貼るか、ホスト型のアイデアページを共有。
3. **登録と順位が集まる** — メールはダブルオプトインで確認。登録者と順位をダッシュボードで管理。

### 3. 開発者にうれしいこと（4つ）
- 📈 **作る前に需要を検証** — 「欲しい人」の数と集まる速さで、リリース前に手応えを掴めます。
- 🧩 **ノーコードで埋め込み** — Web Component を数行貼るだけ。既存の LP やどんなサイトにも。
- ✅ **ちゃんと届くメール** — SPF / DKIM / DMARC 対応・ダブルオプトインで、確認済みの本物だけが残ります。
- 🏅 **順位で熱量を維持** — 登録者は自分の順位を確認できます。早く登録するワクワクが、待ち時間を楽しみに変えます。

### 4. 登録する人にも、ちゃんと価値を（従・短く）
- ウェイトリストに登録した人は、**自分の順位をいつでも確認**でき、**リリース時に通知**が届きます。
  待つ時間が、楽しみに変わります。
  ※（紹介で順位が上がる等のゲーミフィケーションは将来構想。今は書かない）

### 5. 末尾 CTA
- 見出し: **さあ、ウェイトリストを始めよう。**
- 文: 招待コードをお持ちなら、ログインしてすぐにプロジェクトを作成できます。
- CTA: **ログインして始める →**（→ `/login`）
- ※「無料で登録」「ドキュメントを見る」は使わない（ドキュメントページ未整備・招待制のため）。

### 6. フッター
- ロゴ `Launchia` ／ © Launchia
- リンクは**実在するものだけ**: **プライバシーポリシー**（→ `/privacy`）。
  ※ Pricing / Terms / Documentation 等のページは未整備なので、用意できるまでリンクしない。
