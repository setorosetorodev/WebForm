# Launchia (ランシア) - Project Requirements Document (PRD)

## 1. プロジェクト概要 (Overview)
Launchiaは、リリース前のアプリやサービスに特化した**「次世代ウェイトリスト・マーケットプレイス」**です。
開発者にとっては「ローンチ前の熱量を最大化するインフラ」であり、ユーザーにとっては「次に流行るサービスの初期メンバーになれる場所」として機能します。

- **プロジェクト名:** Launchia (ランシア)
- **ドメイン:** launchia.net (取得済)
- **コンセプト:** 熱量の可視化と先行体験の共有

## 3. ターゲットユーザー (Target Audience)

### A. 開発者（Supply Side）
- ローンチ前に初期ユーザー（アーリーアダプター）を獲得したい個人開発者やスタートアップ。
- 自社LP（ランディングページ）に簡単にウェイティングリストを実装したい人。

### B. 一般ユーザー（Demand Side）
- 新しいツールやアプリをいち早く触ることに喜びを感じる層。
- 有名サービスの「初期メンバー（古参）」になりたいアーリーアダプター。

## 4. コア機能 (Core Features)

### 開発者向け機能（SaaSとしての価値）
- **Webパーツ生成:** 1行のJavaScriptタグで、任意の外部サイト（自社LPなど）に高機能なウェイトリストを埋め込み可能。
- **マルチテナント管理:** 1つのアカウントで複数プロジェクトのリストを管理・分析できるダッシュボード。
- **バイラル・エンジン:** 「友人を招待すると順番が繰り上がる」等の紹介ランクシステム。

### 一般ユーザー向け機能（Marketplaceとしての価値）
- **ディスカバリー検索:** 未リリースの魅力的なプロジェクトを探せる一覧ページ。
- **マイ・ダッシュボード:** 自分が登録している全ウェイトリストの「現在の順位」「獲得バッジ」「ステータス」を一元管理。
- **開発者フィード:** 登録したプロジェクトの開発進捗やアップデート通知を受け取る機能。

## 5. 技術アーキテクチャ (Tech Stack)

モダンかつスケーラブルな構成を想定。

- **フロントエンド（管理画面・マーケットプレイス）:** Next.js + Shadcn UI
- **フロントエンド（埋め込みパーツ）:** Web Components (Shadow DOM) によるカプセル化
- **バックエンド API:** Hono + TypeScript (Cloudflare Workers 等でのエッジ実行を想定)
- **データベース:** Neon (PostgreSQL) 
- **DB設計方針:** すべてのコアテーブル（ユーザー、ウェイトリスト等）に `project_id` を持たせ、テナント（プロジェクト）ごとにデータを厳格に分離する設計。

## 6. 開発ロードマップ (Roadmap)

### Phase 1: MVP（最小限のプロダクト）＆ 自社ツール化
- **目的:** システムの根幹となるウェイトリスト機能の確立。
- **実装内容:** 
  - DBスキーマの構築（開発者、プロジェクト、登録者）。
  - APIエンドポイントの作成。
  - 基本的な埋め込み用スクリプトの動作確認。

### Phase 2: マルチテナントSaaS化
- **目的:** 他の開発者が Launchia に登録し、自身のプロジェクトのウェイトリストを作成できる状態にする。
- **実装内容:** 
  - 開発者向け管理ダッシュボード（Next.js）の構築。
  - プロジェクト作成・設定機能。
  - 埋め込みコードの自動生成機能。

### Phase 3: マーケットプレイスの公開
- **目的:** 一般ユーザーがアクセスし、プロジェクトを探せるポータルサイト化。
- **実装内容:** 
  - ランキング、検索、カテゴリ分け機能。
  - ユーザー用マイページの公開（複数プロジェクトの順位確認）。
  - ゲーミフィケーション（バッジや紹介システム）の強化。
