# Launchia プロジェクト戦略・ドメイン計画まとめ (2026-05-27)

## 1. プロジェクトの盛り上げとマネタイズ戦略

### 一般ユーザー向けアプローチ
*   **価値提供:** 「先行体験の特権」と「初期メンバーとしてのステータス」。
*   **盛り上げ策:** 招待制ランキング、限定バッジ、開発者とのクローズドな交流。
*   **収益化:** Launchia Pass（優先登録権）、デジタル記念品（NFT等）の販売。

### 開発者向けアプローチ
*   **価値提供:** 「最小コストでのバイラル・マーケティング・エンジン」。
*   **盛り上げ策:** 成功事例（だすね等）の共有、導入の極限までの簡略化。
*   **収益化:** SaaSモデル（Proプランでの機能拡張）、成功報酬型ブースト（CPA広告）。

## 2. ドメイン・構成戦略

### 2.1 独自ドメインの取得
*   **dasune.net:** アプリ本体として取得。1,000円程度の投資でも、信頼性とバイラル時のURLの美しさ、Launchiaのデモ効果として「買い」と判断。

### 2.2 LP（ランディングページ）の配置構成
*   **構成案:** `setorosetoro.com/dasune` (配下フォルダ形式) を推奨。
*   **理由（SEO）:** アクセス評価や被リンクの価値を親ドメイン `setorosetoro.com` に集約し、ドメインパワーを強化できるため。
*   **理由（AdSense）:** 一度の審査で配下ページすべてに広告を掲載でき、管理効率と収益性が高いため。

## 3. Launchia MVP 開発の優先順位

1.  **コア機能の作成:** 既存の `company-contact-form.ts` をベースに、ウェイトリスト専用の埋め込みパーツ（Web Components）を開発。
2.  **ドッグフーディング:** 取得した `dasune.net`（またはポートフォリオ内のLP）にパーツを導入し、自社プロジェクト「だすね」で運用実績を作る。
3.  **ポータル公開:** 実績（数値）を携えて、Launchia自体のマーケットプレイスを公開し、他の開発者へ展開する。
